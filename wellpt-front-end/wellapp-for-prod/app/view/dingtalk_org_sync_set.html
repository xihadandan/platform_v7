配置从钉钉同步至OA的规则，开关开启为同步，开关关闭为不同步

1、开关关闭：不同步，保留OA信息，适用无法满足OA功能的钉钉信息

2、开关开启：同步，以钉钉信息为准，适用完全满足OA功能的钉钉信息

人员和职位的关系
建议： 同步
建议理由：满足企业人员较少【一人多岗】的通用场景
  • 人员在钉钉上的部门只有1个：同步后即为OA的主职位
  • 人员在钉钉上的部门有多个时：需要人工审核该人员在OA的职位，审核后才会在OA上生效
钉钉和OA的职位存在以下差异
1、钉钉
  • 部门：支持多个，属于组织架构
  • 职位：只有1个，不属于组织架构
2、OA
  • 部门：职位归属的部门，属于组织架构
  • 职位：=主职位（只有1个）+其他职位（支持多个），有职位时，一定有主职位；属于组织架构
