# 工程用途
存放平台内置的定义文件或者sql文件。

# 文件及夹说明
- 该工程的根目录下一般存放的是以版本号命名的文件夹,这里称之为版本文件夹。
- 在版本文件夹下存放平台内置的定义文件或者sql文件。

## 文件夹命名规范
该工程下的文件夹一般以版本号作为文件夹名称

## 文件命名规范
格式如下: 
```
${序号}_${执行位置}_${模块名}_${文件用途说明}.${文件扩展名}
```  

### 格式说明：
- 序号： 用于表达该文件执行的顺序，数值越小越先执行

    序号由三位数据组成，不足三位在前面补0
    
- 执行位置： 用于表达该文件执行的位置, 该值有四种值，请参照下面描述:

			admsql: 公共库sql
	
			usrsql: 租户库sql
	
			superdef: 超级管理配置
	
			unitdef: 系统单位配置
	
- 模块名： 该文件所属模块

- 文件用途说明： 用于描述该文件的用途

### 示例： 
	
	```
	    010_admsql_消息管理_创建消息分类表.sql
	```
	
	```
	    011_superdef_消息管理_消息格式列表.dfd
	```
