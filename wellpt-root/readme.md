
# 一、序言

## 平台模块循环依赖解决方案

平台模块之间如存在循环依赖，这时，如何解决?
1. 解决方案为:通过依赖注入。
模块对外暴露接口，模块之间使用接口进行交互， 再通过控制反转的方式将实现注入即通过依赖注入的方式来解决.

2. 存在循环依赖的模块需要分成两个jar, 一个是提供给外部使用的接口，一个为模块实现

模块接口对应的jar包的命名规范为: **wellpt-${模块名称}-support**,  
模块实现对应的jar包的命名规范为: **wellpt-${模块名称}-implement**


# 二、目录结构说明
## 1. wellpt-modules
pom工程， 该目录本身没有体现为功能, 用于存放平台的各模块
注:
该目录下的模块主要有两类:
1. 完整的模块，该类模块不存在循环依赖, 其他模块可以直接依赖于该jar工程
2. implement类模块，即为为解决循环依赖而产生的implement模块，如果需要依赖该功能模块，则依赖对应的support模块

## 2. wellpt-module-supports
pom工程， 该目录本身没有体现为功能, 用于存放解决循环依赖而产生的support工程

## 3. wellpt-3rd-libs
pom工程， 用于存放已被平台侵入的开源框架

## 4. wellpt-webs
pom工程， 用于存放web项目
